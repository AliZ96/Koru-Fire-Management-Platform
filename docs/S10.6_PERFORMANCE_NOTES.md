# S10.6 - System Performance Optimization Notes

Bu dokuman, story kapsaminda uygulanan performans iyilestirmelerini ve operasyonel guardrail'leri ozetler.

## Uygulanan Iyilestirmeler

- API request timing middleware eklendi (`app/main.py`)
  - Her response icin `X-Response-Time-Ms` header'i yazilir.
  - 800ms ustu istekler `app.performance` logger ile "slow request" olarak loglanir.

- Static GeoJSON yukleme cache'lendi (`app/api/routers/core.py`)
  - `load_geojson()` fonksiyonu `lru_cache` ile tekrarli disk I/O maliyetini azaltir.

- Fire risk endpoint cache + pagination guardrail (`app/api/routers/fire_risk.py`)
  - CSV yukleme `lru_cache(maxsize=1)` ile cache'lendi.
  - `limit` icin default azaltildi (10000), `offset` destegi eklendi.
  - Cevap metadatasina `offset`, `limit`, `total_available` eklendi.

- Dis API cagri cache katmani eklendi
  - FIRMS: `app/services/firms_service.py` (TTL, env: `FIRMS_CACHE_TTL_SECONDS`, default 120s)
  - Weather: `app/services/weather_service.py` (TTL, env: `WEATHER_CACHE_TTL`, default 300s)

- Routing graph olusturma optimize edildi (`app/services/routing_service.py`)
  - LOW_RISK node sayisina ust sinir ve ornekleme eklendi (`ROUTING_MAX_LOW_NODES`, default 1200).
  - O(n^2) komsuluk aramasi yerine spatial bucket tabanli aday daraltma kullanildi.
  - Istasyon->LOW, LOW->LOW, LOW->HIGH kenar olusumu bucket odakli hesaplanir.

- Grid tabanli endpointler icin request guardrail
  - `app/api/routers/air_accessibility.py`: min grid size + max tahmini nokta sayisi limiti.
  - `app/api/routers/resource_proximity.py`: min cell size + bbox alan limiti.
  - `app/api/routers/integrated_layer.py`: min cell size + bbox alan limiti.

## Operasyonel Notlar

- Rota kalitesi ve performans arasinda denge:
  - `ROUTING_MAX_LOW_NODES` degeri arttikca rota fidelity artar, graph build suresi de artar.
  - Uretim ortami icin veri buyuklugune gore tuning yapilmalidir.

- Caching davranisi:
  - Cache in-memory oldugu icin process restart'ta sifirlanir.
  - Yatay olcekleme varsa node bazli cache farkliliklari normaldir.

## Onerilen Sonraki Adimlar

- `pytest` ve benchmark araclarini CI ortamina ekleyip p95 latency regression gate tanimla.
- Prometheus benzeri exporter ile endpoint bazli latency/hata metrikleri dashboard'a tasinsin.
- Mobil tarafta bbox/tile bazli kademeli katman yukleme ile buyuk payload etkisi azaltilsin.
